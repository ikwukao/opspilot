## Versioning Policy

OpsPilot follows **Semantic Versioning**.

* **MAJOR** version: incompatible API or behavioral changes.
* **MINOR** version: new functionality added in a backward-compatible manner.
* **PATCH** version: backward-compatible bug fixes.

---

## Release Philosophy

Each release should:

* Deliver meaningful user value.
* Improve reliability.
* Preserve a consistent user experience.
* Maintain high engineering standards.
* Include updated documentation.
